# Risk Register

This file lists findings that future AI agents should account for before changing behavior.

| Priority | Area | Evidence | Why It Matters | Recommended Fix |
| --- | --- | --- | --- | --- |
| P0 | Installed startup | `pyproject.toml:49`, `libercode/__main__.py:1`, `libercode/__main__.py:6` | The installed `libercode` command starts a TUI without attaching `LiberAgent`; chat and agent-backed commands can fail with "Agent not connected." | Route the entry point through `cli.main()` or make `__main__.py` create `LiberAgent` and call `app.set_agent(agent)`. Add an entry-point smoke test. |
| P0 | CLI contract | `libercode/cli.py:202`, `libercode/cli.py:223`, `libercode/cli.py:233`, `libercode/cli.py:240` | `exec`, `config`, and `show` exist but are not reachable through the installed script as documented. | Decide whether `libercode` is argparse-first or TUI-first. Wire both deliberately and test `libercode --version`, `libercode exec`, and default startup. |
| P1 | Config persistence | `libercode/config.py:9`, `libercode/config.py:150`, `libercode/config.py:163`, `libercode/config.py:189`, `libercode/agent.py:1797`, `docs/providers.md:40` | YAML is the main config, while newer provider/config code writes TOML. Current env has no `tomli_w` or `toml`, and the TOML file is not read by `LiberConfig.load()`. | Standardize on YAML or add TOML as a real supported format. Remove silent save failures and add provider save/load tests. |
| P1 | Missing TUI command | `libercode/agent.py:847` | `/memory` is advertised in `tui.py` but calls `_tui_cmd_memory`, which does not exist. | Implement `_tui_cmd_memory` or route to an existing shared command service. Add a TUI dispatch unit test. |
| P1 | PR creation | `libercode/agent.py:1635`, `libercode/agent.py:1642` | `gh pr create` is sent through a Git helper that prefixes `git`, so the command is likely wrong. | Run GitHub CLI with a dedicated subprocess helper or add a `GitHubHelper`. Avoid shell splitting for quoted bodies. |
| P1 | Shell safety | `libercode/shell.py:49`, `libercode/shell.py:78` | `shell=True` plus a substring denylist is brittle. It blocks some legitimate commands while missing many dangerous variants. | Prefer argument-list execution for known commands. For arbitrary shell mode, require explicit user confirmation and improve policy boundaries. |
| P1 | File containment | `libercode/shell.py:80`, `libercode/shell.py:92`, `libercode/agent.py:372`, `libercode/agent.py:387` | Path checks live mostly in `LiberAgent`; direct `ShellExecutor` file methods can traverse. `_read_file` reads before validating. | Move strict `resolve().is_relative_to(workdir)` checks into `ShellExecutor` before read/write/list/grep. |
| P1 | Mode drift | `libercode/agent.py:285`, `libercode/agent.py:345`, `libercode/agent.py:1198`, `libercode/config.py:254`, `libercode/ui.py:31` | `debug` is supported in some surfaces but omitted in others. | Define one `VALID_MODES` constant and import it everywhere. Update prompts, UI colors, wizard choices, and tests. |
| P2 | Giant modules | `libercode/agent.py`, `libercode/tui.py` | These two files dominate the codebase and mix concerns, making changes risky and tests narrow. | Extract command handlers, provider management, TUI modals, shell/file tools, and session operations into smaller modules. |
| P2 | Lint debt | `ruff check .` | 91 findings create noise and hide real defects. | Run safe auto-fixes, then manually fix remaining F821/E701/F811/F841 issues. Add lint to CI. |
| P2 | Checkpoint restore asymmetry | `libercode/agent.py:573`, `libercode/agent.py:1310` | TUI restore validates paths; legacy `/undo` writes files without the same containment check. | Use one shared restore function with path validation. |
| P2 | Branch validation | `libercode/git_utils.py:6`, `tests/test_git_utils.py:19` | Regex validation allows invalid Git ref patterns; one test passes due to non-repo failure, not validation. | Use `git check-ref-format --branch` or a stricter validator. Test inside an initialized temp repo. |
| P2 | Silent exceptions | Many `except Exception: pass` in `agent.py`, `tui.py`, `checkpoint.py`, and `stop_condition.py` | Failures can disappear, especially in provider switching, TUI UI updates, and checkpoint capture. | Log debug details behind a verbose flag or route warnings to the UI/status area. |
| P3 | Workspace debris | `MagicMock/` ignored by `.gitignore` | This is not product code, but it can confuse agents scanning the filesystem. | Keep ignored, or clean periodically outside product changes. |
